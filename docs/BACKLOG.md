# Backlog: направления развития

Крупные инициативы, которые предстоит сделать. Это **эскизы**, а не спеки — цель файла «не потерять и задать приоритет». Когда до пункта дойдут руки, он проходит brainstorming и порождает свою спеку в `docs/superpowers/specs/`.

Приоритеты справочные — порядок работ выбирается по ситуации.

---

## 1. Ревизия всего проекта

Полный аудит кодовой базы: границы легаси/SDUI, качество, соответствие правилам.

- **Зачем:** даёт карту — что мигрировать, что покрывать тестами, где нарушены правила изоляции. Питает остальные направления.
- **Что входит:**
  - Аудит соблюдения границы легаси/SDUI (прямые импорты в обе стороны запрещены — проверить фактически).
  - Проверка FSD-правил: barrel-экспорты только на уровне слайсов, лимиты строк на новый код (~200, >300 разбить).
  - Инвентаризация легаси, готового к удалению после миграции.
  - Свести с существующими аудитами SDUI (`specs/2026-07-02-sdui-course-audit.md`, `specs/2026-07-02-sdui-code-review.md`) и расширить на весь проект.
- **Зона:** весь проект (легаси + SDUI + shared).
- **Приоритет:** высокий.
- **Статус:** не начато.
- **Следующий шаг:** brainstorming → спека в `specs/`.

---

## 2. Тесты для ключевых модулей

Покрыть тестами критичные места перед активной миграцией легаси→SDUI.

- **Зачем:** страховка от регрессий при рефакторинге и миграции.
- **Что входит:**
  - Выбрать и настроить фреймворк (вероятно Vitest — под Vite).
  - Определить «ключевые модули» (кандидаты: SDUI-рендерер, form-renderer, EAV-lib, filter-lib, workspace-tabs).
  - Тесты на бизнес-логику, а не на разметку.
- **Зона:** общее + SDUI (по итогам ревизии уточнить приоритеты).
- **Приоритет:** высокий.
- **Статус:** не начато.
- **Следующий шаг:** brainstorming → спека в `specs/`.

---

## 3. Синхронизация дизайна и UI-либы с Figma

Единый источник правды для компонентов и токенов между Figma и кодом.

- **Зачем:** убрать расхождения дизайн ↔ реализация, ускорить разработку UI.
- **Что входит:**
  - Сверка UI-компонентов (`shared/ui/`, MUI + Tailwind) с Figma-библиотекой.
  - Дизайн-токены (цвета, типографика, отступы) как общий источник.
  - Оценить Code Connect (Figma MCP уже подключён) для маппинга компонентов.
- **Зона:** общее (`shared/ui/`).
- **Приоритет:** средний.
- **Статус:** не начато.
- **Следующий шаг:** brainstorming → спека в `specs/`.

## 4. Чат-бот для синхронизации спек

Автоматизация workflow «спеки в комментариях Jira» (обмен фронт↔бэк).

- **Зачем:** сейчас передача спек между фронтом и бэком ручная (файл → вложение → комментарий → перенос колонки). Бот снимает рутину.
- **Что входит:**
  - Формализовать текущий цикл передачи (см. `CLAUDE.local.md` → «Workflow "Спеки в Jira"»).
  - Определить триггеры и действия бота (уведомления о новой версии спеки, автоперенос колонок, напоминания).
  - Выбрать платформу/интеграцию.
- **Зона:** инфраструктура (вне продуктового кода).
- **Приоритет:** низкий.
- **Статус:** не начато.
- **Следующий шаг:** brainstorming → спека в `specs/`.
