# Copy Entry API — Backend Specification

Спецификация для реализации эндпоинтов копирования записей справочников и документов на Java Spring Boot.

## Эндпоинты

### 1. Копирование записи справочника

```
POST /api/universaldomain-entries/{domain}/id/{id}/copy
```

**Path Parameters:**
| Параметр | Тип | Описание |
|----------|-----|----------|
| `domain` | string | Домен справочника (например: `DICTIONARY`, `ACCUMULATION_REGISTER`, etc.) |
| `id` | long | ID исходной записи для копирования |

**Request Body:** отсутствует

**Response:** `201 Created`
```json
{
  "data": {
    "id": 42,
    "code": "COPY_OF_ABC123",
    "nameRu": "Копия — Название записи",
    "nameKz": "Көшірме — Жазба атауы",
    "displayName": "Копия — Название записи",
    "isActive": true,
    "attributes": { ... }
  },
  "success": true
}
```

**Логика:**
1. Найти исходную запись по `domain` и `id`
2. Если запись не найдена → `404 Not Found`
3. Создать новую запись, скопировав все поля:
   - `id` — новый (автогенерация)
   - `code` — сгенерировать новый уникальный код (например, `{originalCode}_COPY` или `{originalCode}_COPY_1`)
   - `nameRu` — `"Копия — {originalNameRu}"`
   - `nameKz` — `"Көшірме — {originalNameKz}"` (если есть)
   - `parentId` — скопировать из исходной
   - `sortOrder` — скопировать из исходной
   - `isActive` — `true`
   - `attributes` — полная копия всех атрибутов исходной записи
4. Сохранить новую запись в БД
5. Вернуть созданную запись в response

**Ошибки:**
| Код | Описание |
|-----|----------|
| `404` | Запись с указанным ID не найдена |
| `500` | Внутренняя ошибка сервера |

---

### 2. Копирование записи документа

```
POST /api/document-entries/{typeCode}/{id}/copy
```

**Path Parameters:**
| Параметр | Тип | Описание |
|----------|-----|----------|
| `typeCode` | string | Код типа документа |
| `id` | long | ID исходного документа для копирования |

**Request Body:** отсутствует

**Response:** `201 Created`
```json
{
  "data": {
    "id": 99,
    "documentTypeCode": "INVOICE",
    "code": "INV-000042",
    "nameRu": "Копия — Счёт №123",
    "nameKz": "Көшірме — Шот №123",
    "parentId": null,
    "sortOrder": 0,
    "isActive": true,
    "isPosted": false,
    "attributes": { ... },
    "children": [],
    "createdAt": "2026-05-11T12:00:00",
    "updatedAt": "2026-05-11T12:00:00",
    "deletedAt": null
  },
  "success": true
}
```

**Логика:**
1. Найти исходный документ по `typeCode` и `id`
2. Если документ не найден → `404 Not Found`
3. Создать новую запись, скопировав поля:
   - `id` — новый (автогенерация)
   - `documentTypeCode` — скопировать из исходного
   - `code` — сгенерировать новый уникальный номер (по правилам нумерации типа документа)
   - `nameRu` — `"Копия — {originalNameRu}"`
   - `nameKz` — `"Көшірме — {originalNameKz}"` (если есть)
   - `parentId` — скопировать из исходного
   - `sortOrder` — скопировать из исходного
   - `isActive` — `true`
   - **`isPosted` — `false`** (копия всегда не проведена)
   - `attributes` — полная копия всех атрибутов
   - `children` — скопировать дочерние записи (табличные части), каждая с новым ID
   - `createdAt` / `updatedAt` — текущее время
   - `deletedAt` — `null`
4. Сохранить новую запись и все дочерние записи в БД
5. Вернуть созданную запись в response

**Важно:**
- Копия документа **всегда `isPosted: false`** — нельзя создавать проведённые документы через копирование
- Дочерние записи (табличные части / children) должны быть скопированы вместе с основной записью
- Номер (code) генерируется по тем же правилам, что и при создании нового документа

**Ошибки:**
| Код | Описание |
|-----|----------|
| `404` | Документ с указанным ID и typeCode не найден |
| `500` | Внутренняя ошибка сервера |

---

## Общие требования

- HTTP метод: `POST` (создаёт новый ресурс)
- Response code при успехе: `201 Created`
- Response format: `{ "data": {...}, "success": true }` — стандартный `ApiResponse<T>` обёртка проекта
- Оба эндпоинта не принимают request body
- Операция атомарная — если не удалось скопировать дочерние записи, откатить всю транзакцию

## Примеры cURL

```bash
# Копирование записи справочника
curl -X POST http://92.38.49.213:31880/api/universaldomain-entries/DICTIONARY/id/15/copy

# Копирование документа
curl -X POST http://92.38.49.213:31880/api/document-entries/INVOICE/42/copy
```
