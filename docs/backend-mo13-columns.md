# Backend: `ReportFormDto` для МО-13 / Форма 396 — полный набор колонок под 1С

> Спека к тикету «3-уровневая шапка и графы бланка 1-в-1 с 1С».
> Фронт (layout=FORM) уже generic: рисует все колонки из `form.sections[].columns[]`,
> строит 1/2/3-уровневую шапку по `groupTitleRu`/`subGroupTitleRu`, применяет
> `width`, `align`, `format`, `blankOnZero`. Число и состав колонок задаёт **только бэк**.

## Порядок деплоя

Сначала **фронт**, потом **бэк**. Новый фронт терпит старый бэк (2-уровневая шапка, авто-ширины,
без падений); обратный порядок даёт пользователям окно «плоской» шапки на старом фронте.

## Проблема

DTO присылает меньше граф-субсчетов, чем печатная форма 1С: в вебе ряд нумерации `1..14`,
в 1С — `1..16`. Надо привести число и состав колонок к 1С точь-в-точь.

## Эталон (1С, Форма 396) — порядок колонок слева направо

| Граф | Ряд 1 `groupTitleRu` | Ряд 2 `subGroupTitleRu` | Ряд 3 `titleRu` | role | align | Примечание |
|---|---|---|---|---|---|---|
| 1 | — | — | `№ п/п` | ATTRIBUTE | LEFT | на всю высоту шапки |
| 2 | — | — | `Дата` | PERIOD | LEFT | format `dd.MM.yyyy` |
| 3 | — | — | `Номер документа` | ATTRIBUTE | LEFT | |
| 4 | — | — | `Наименование документа` | ATTRIBUTE | LEFT | |
| 5…K | `Кредит субсчетов` | — | `<субсчёт>` (напр. `1315`) | MEASURE | RIGHT | по колонке на каждый кредитуемый субсчёт + зарезервированные пустые до фикс. числа формы |
| K+1 | — | — | `Итого` | MEASURE | RIGHT | итог по кредиту, на всю высоту |
| …пары | `Дебет субсчетов` | `<субсчёт>` (напр. `7060`) | `специфика` | ATTRIBUTE | LEFT | по паре (специфика, сумма) на каждый дебетуемый субсчёт |
| …пары | `Дебет субсчетов` | `<субсчёт>` | `сумма` | MEASURE | RIGHT | |
| M+1 | — | — | `Итого` | MEASURE | RIGHT | итог по дебету, на всю высоту |
| …N | `В том числе по государственным учреждениям (материально-ответственным лицам)` | — | `<МОЛ>` (напр. `Иванова В.С.`) | MEASURE | RIGHT | по колонке на каждого МОЛ + зарезервированные пустые |

## Правила формирования колонок

1. **Порядок строгий**: id-колонки → графы `Кредит субсчетов` → `Итого`(кредит) →
   пары `Дебет субсчетов` → `Итого`(дебет) → графы `В том числе…`.
2. **Фиксированное число слотов** как в печатной форме 396 (по наблюдению в 1С — по 5 граф
   на «Кредит субсчетов» и «Дебет субсчетов»; уточнить по бланку). Незадействованный субсчёт —
   колонку всё равно вернуть: `titleRu=""`, ячейки пустые. Фронт нарисует пустую графу с рамкой
   и номером в ряду нумерации — 1-в-1 с 1С.
3. **`subGroupTitleRu` только на парах «Дебет субсчетов»** — номер счёта над `специфика`/`сумма`.
   У «Кредит субсчетов» и «В том числе…» — `groupTitleRu` задан, `subGroupTitleRu = null`,
   а номер субсчёта/ФИО кладётся в `titleRu` (фронт растянет его на ряды 2–3).
4. **`Итого` и id-колонки**: `groupTitleRu = null`, `subGroupTitleRu = null` → фронт даёт им
   rowSpan на всю высоту шапки.
5. **Ширины**: `width` в символах (фронт: ≈ width×8px). Под 1С: id узкие ~4–10, наименование ~24,
   суммовые ~16, чтобы значения не обрезались.
6. **Форматы/поведение**: суммовые — `format:"#,##0.00"`, `align:"RIGHT"`, `blankOnZero:true`;
   текстовые — `align:"LEFT"`.

## Пример фрагмента `columns` (Дебет с двумя субсчетами)

```jsonc
{ "code":"Debet_7060_spec", "groupTitleRu":"Дебет субсчетов", "subGroupTitleRu":"7060",

  "format":"#,##0.00", "blankOnZero":true },
{ "code":"Debet_7010_spec", "groupTitleRu":"Дебет субсчетов", "subGroupTitleRu":"7010",
  "titleRu":"специфика", "role":"ATTRIBUTE", "align":"LEFT", "width":16 },
{ "code":"Debet_7010_sum",  "groupTitleRu":"Дебет субсчетов", "subGroupTitleRu":"7010",

  "format":"#,##0.00", "blankOnZero":true }
```

## Дополнительно (для полного совпадения)

- **Верхняя строка-остаток** (в 1С сразу под рядом нумерации: `600 000,00 … 600 000,00`,
  без подписи) — вернуть отдельной строкой в `sections[].rows` (напр. `rowKind:"TURNOVER"`/opening,
  `labelText:""`, значения в `cells`). Фронт её отрисует.
- **`«ВСЕГО:»`** — это `labelText` строки-итога (`rowKind:"TOTAL"`); сейчас приходит `«Всего»`.
  Привести текст к 1С при необходимости.
- **Заголовок группы МОЛ** — присылать полностью:
  `«В том числе по государственным учреждениям (материально-ответственным лицам)»`
  (сейчас без скобочной части).

## Критерии приёмки

- Ряд нумерации граф и число колонок в вебе совпадают с 1С (для того же периода/счетов).
- «Кредит субсчетов» и «Дебет субсчетов» — фиксированное число граф как в форме 396;
  незадействованные графы пустые, но присутствуют.
- «специфика/сумма» повторяются парами по числу дебетуемых субсчетов.
- Значения не обрезаются (корректные `width`), суммы — `450 000,00`.
- Веб-бланк совпадает с кнопкой «Печать» и с 1С.

## Что уже сделано на фронте (generic, деплоить первым)

- `ReportColumnDto`: добавлены опциональные `subGroupTitleRu`/`subGroupTitleKz`, `width`.
- Рендерер бланка: 1/2/3-уровневая шапка по наличию `groupTitleRu`/`subGroupTitleRu`
  (colSpan/rowSpan generic, без хардкода под МО-13); номер счёта-подгруппы — жирный курсив.
- Ширины колонок из `width` (≈ ×8px), суммы в RU-формате, `blankOnZero`, `align`.
- Полноширинный лист + горизонтальный скролл (широкий бланк не режется и не ломает страницу;
  на печати разворачивается целиком).
