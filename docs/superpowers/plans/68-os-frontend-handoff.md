# Задачи для фронтенд-разработчика — карточка ОС и Поступление от контрагента (домен ОС)

Три задачи по замечаниям аналитика. По каждой: что сейчас не так → как должно быть (по эталону 1С) → что сделать → как проверить. Бэкенд для них уже готов (или изменения бэкенда не требуется — это UI/форма).

---

## Задача 1 — «Вид операции» должен подставляться автоматически

**Документ:** Поступление от контрагента.

**Что сейчас не так:** при создании нового документа показывается диалог «Выберите операцию» (Поступление от контрагента / Безвозмездное поступление / … 7 вариантов). Пользователь выбирает операцию, но новый документ открывается с **пустым** полем «Вид операции» (а поле обязательное со звёздочкой). В 1С это поле сразу заполнено выбранной операцией.

**Как должно быть (1С):** выбранная в диалоге операция подставляется в поле «Вид операции» нового документа.

**Что сделать:** при открытии нового документа из диалога передавать выбранную операцию как параметр открытия формы `ВидОперации` (`VidOperatsii`). Бэкенд уже это принимает и проставит поле сам — нужно только, чтобы фронт **передал выбранное значение** в параметрах открытия. Если параметр не передан — поле останется пустым (как сейчас).

**Технический ориентир:** значение = код перечисления из типа `VidyOperatsiyPostuplenieOtKontragenta`. Семь значений:
`PostuplenieOtKontragenta`, `BezvozmezdnoePostuplenie`, `PostupleniePoRezultatamInventarizatsii`, `PostuplenieObektovStroitelstva`, `OtvetstvennoeKhranenie`, `BlankiStrogoyOtchetnosti`, `PostupleniePoRezultatamRazukomplektatsii` (это те же 7 пунктов из диалога, по порядку).

**Как проверить:** создать документ через диалог, выбрать любую операцию → новый документ открывается с уже заполненным полем «Вид операции» = выбранная операция.

## Задача 2 — «Группа ОС» в карточке ОС: показывать только группы, а не весь список

**Справочник:** Основные средства (карточка/создание).

**Что сейчас не так:** в поле «Группа ОС» выпадающий список показывает **весь плоский справочник** «Виды долгосрочных активов» — сотни позиций (Автомобильные дороги, Авторские права, Библиотечный фонд и т.д.). Это неудобно.

**Как должно быть (1С):** в поле «Группа ОС» выбираются только **группы (папки)** — короткий список примерно из 13 позиций:

| Код счёта | Группа |
|---|---|
| 2310 | Земля |
| 2320 | Здания |
| 2330 | Сооружения |
| 2340 | Передаточные устройства |
| 2350 | Транспортные средства |
| 2360 | Машины и оборудование |
| 2370 | Инструменты, производственный и хозяйственный инвентарь |
| 2380 | Прочие основные средства |
| 2510 | Инвестиционная недвижимость |
| 2600 | Биологические активы |
| — | Незавершённое строительство и капитальные вложения |
| — | Нематериальные активы |
| — | Прочие долгосрочные активы |

**Что сделать:** пикер поля «Группа ОС» должен показывать только **папки-группы** справочника «Виды долгосрочных активов», а не конечные элементы. (В 1С это настройка поля «выбор только групп».) Тип поля НЕ менять — это по-прежнему выбор из справочника «Виды долгосрочных активов», просто ограниченный уровнем групп.

**Технический ориентир:** справочник «Виды долгосрочных активов» в системе иерархический (папки + элементы). У эндпоинта дерева справочника есть параметр `groupsOnly` — использовать его для этого пикера (показывать только группы/папки). Коды 2310–2600 — это «Счёт учёта» соответствующей группы.

**Как проверить:** открыть карточку ОС → нажать выбор в поле «Группа ОС» → показывается короткий список групп (как в таблице выше), а не длинный плоский список всех видов. Выбранная группа сохраняется.

## Задача 3 — Убрать поле «Вид ОС» из карточки ОС

**Справочник:** Основные средства (карточка/создание).

**Что сейчас не так:** в карточке ОС есть поле «Вид ОС» (сразу под «Группа ОС»). В 1С этого поля на форме нет.

**Как должно быть (1С):** поле «Вид ОС» на карточке **скрыто** (в 1С у него стоит «Видимость = ложь»). Само хранилище можно не трогать — достаточно убрать поле с экрана.

**Что сделать:** скрыть/убрать поле «Вид ОС» с формы карточки ОС.

**Как проверить:** открыть карточку ОС → поля «Вид ОС» на форме больше нет.

---

## Примечание
Задачи 2 и 3 — чисто про отображение формы карточки ОС (карточка рисуется фронтом, в бэкенде для этих полей отдельной настройки нет). Задача 1 — бэкенд уже готов принять параметр, нужен только проброс выбранной операции с фронта. Никакой перезагрузки/миграций для этих трёх задач не требуется — только выкладка фронта.
