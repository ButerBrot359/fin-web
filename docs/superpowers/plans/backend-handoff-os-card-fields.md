# Задачи для бэкенда — карточка ОС и «Поступление от контрагента» (домен ОС)

Постановка по итогам ревизии замечаний аналитиков
([исходная постановка](68-os-frontend-handoff.md),
[дизайн-спека](../specs/2026-07-03-os-card-fields-design.md)).
Проверка выполнялась на dev-стенде `https://dev-api.qazyna.ai` 2026-07-03.

Принятое решение: настройки полей карточки живут в **метаданных типа** (как в 1С),
фронт реагирует на них универсально. Для этого от бэкенда нужны три изменения ниже.
Фронт свою часть выкладывает независимо — до готовности бэка поведение не меняется.

---

## Задача 1 — «Группа ОС»: `referenceSelectionMode = GROUP` в метаданных

**Что сейчас:** у атрибута `GruppaOS` типа `OsnovnyeSredstva`
(`GET /api/universaldomain-types/DICTIONARY/OsnovnyeSredstva`)
`referenceSelectionMode: "GROUP_AND_ELEMENT"`.

**Что сделать:** выставить `referenceSelectionMode: "GROUP"` (в 1С это настройка
поля «выбор групп»). Ничего другого в атрибуте не менять — тип остаётся ссылкой
на справочник «Виды долгосрочных активов».

**Как это использует фронт:** увидев `referenceSelectionMode === "GROUP"` у
ссылочного атрибута, фронт добавляет `groupsOnly=true` в запросы пикера этого поля
(см. задачу 2). Работает для любого атрибута любого типа — без правок фронта в будущем.

**Если в enum нет значения `GROUP`** (используется другой литерал, например
`ONLY_GROUP`) — сообщить фронту фактическое значение, матчинг на фронте
подстроят под него.

---

## Задача 2 — параметр `groupsOnly` у search/paged универсального домена

**Что сейчас:** фильтрация по группам не работает у эндпоинтов, которыми ходит
пикер и список «Показать все»:

```
GET /api/universaldomain-entries/DICTIONARY/VidyDolgosrochnykhAktivov/search?q=&size=1000
→ 64 записи (группы и элементы вперемешку)

GET .../search?q=&size=1000&groupsOnly=true
→ те же 64 записи — параметр игнорируется
```

Замечание: у эндпоинта дерева `GET /api/universaldomain-directories/...`
`groupsOnly=true` работает (возвращает 13 групп), но пикер фронта ходит
не в него, а в `/search` (нужен плоский поиск по подстроке), и «Показать все» —
в `/paged`. Поэтому поддержка нужна именно там.

**Что сделать:** научить оба эндпоинта параметру `groupsOnly`:

```
GET /api/universaldomain-entries/{domain}/{typeCode}/search?q=...&groupsOnly=true
GET /api/universaldomain-entries/{domain}/{typeCode}/paged?page=...&groupsOnly=true
```

- `groupsOnly=true` → в выдаче только записи с `isGroup = true` (на всех уровнях
  иерархии); поиск `q`, пагинация и прочие фильтры (`af` и т.д.) работают как обычно.
- Параметр не передан или `false` → текущее поведение без изменений.

**Как это использует фронт:** пикер поля с `referenceSelectionMode = GROUP`
шлёт `groupsOnly=true` и в `/search` (выпадающий список), и в `/paged`
(панель «Показать все»).

**Как проверить:**

```
GET /api/universaldomain-entries/DICTIONARY/VidyDolgosrochnykhAktivov/search?q=&size=100&groupsOnly=true
```

→ ровно 13 записей, все `isGroup: true`: Земля, Здания, Сооружения, Передаточные
устройства, Транспортные средства, Машины и оборудование, Инструменты…,
Прочие основные средства, Инвестиционная недвижимость, Биологические активы,
Незавершённое строительство…, Нематериальные активы, Прочие долгосрочные активы.

## Задача 3 — скрыть «Вид ОС»: `showInForm = false` в метаданных

**Что сейчас:** у атрибута `VidOS` типа `OsnovnyeSredstva` — `showInForm: true`,
поэтому фронт рендерит поле на карточке.

**Что сделать:** выставить `showInForm: false` (аналог 1С «Видимость = ложь»).
Само хранилище/данные атрибута не трогать.

**Как это использует фронт:** карточка ОС строится из атрибутов с
`showInForm: true` — поле исчезнет с формы без изменений кода фронта.

**Как проверить:** открыть карточку ОС → поля «Вид ОС» нет; в списке (если
`showInList` не менялся) колонка остаётся.

## Не требует работ: «Вид операции» в «Поступление от контрагента»

Проверено end-to-end 2026-07-03: диалог «Выберите операцию» → фронт передаёт
`VidOperatsii={код операции}` → `GET /api/document-entries/PostuplenieOtKontragenta/new?VidOperatsii=BezvozmezdnoePostuplenie`
возвращает `attributes.VidOperatsii` заполненным → форма открывается с
заполненным «Вид операции». Задача 1 из постановки аналитиков закрыта —
достаточно перепроверить на их стенде.
