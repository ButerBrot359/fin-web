## 2. Ревью кода

Только чтение и отчёт, никаких правок. Два фокуса:

### Фокус 1 — качество SDUI-кода
- Структура движка: registry, dispatch, рендереры узлов.
- Размеры и связность файлов (файлы-переростки, смешение ответственностей).
- Дублирование кода, мёртвый код.
- Типизация: `any`, небезопасные касты, дыры в типах SDUI-схемы.
- Соответствие FSD: barrel-экспорты только на уровне слайсов, api/ для запросов.
- Консистентность паттернов: useMutation для мутаций, отсутствие
  useMemo/useCallback без явной необходимости, i18n-ключи вместо хардкода.

### Фокус 2 — граница легаси ↔ SDUI
Главное требование: легаси и SDUI взаимонезависимы и не конфликтуют.
- Карта общих модулей: кто импортирует кого (SDUI → легаси, легаси → SDUI, оба → shared).
- Точки ветвления между путями (`newView=true` и подобные).
- Общий state, кэш TanStack Query, общие типы.
- Риски «поправил легаси — сломал SDUI» и наоборот, с конкретными файлами.
