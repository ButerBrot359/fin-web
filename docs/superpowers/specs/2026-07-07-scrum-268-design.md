# SCRUM-268 (+хвост SCRUM-218) — дизайн реализации во fin-web

Дата: 2026-07-07. Ветка: `feat/scrum-268` (от `dev`, tip = merge PR #210).
Источник требований: `docs/superpowers/plans/SCRUM-268-frontend-spec.md` (бэкенд-контракт, ветка `talgat/SCRUM-268`).
Связано: `docs/superpowers/specs/2026-07-02-sdui-course-audit.md` (канон; раздел 9 — отклонения этой волны).

## Цель

Реализовать во fin-web полный объём спеки SCRUM-268 §3.1–3.7: составной OBJECT-виджет,
живое дерево drawer'а, download attachment/inline, редирект-роуты `/documents/*`,
общий хук загрузки опций, i18n.

## Границы

- **Только SDUI-путь** (типы документов с `newView=true`) + роуты в `App.tsx`.
- **Легаси form-renderer не трогаем вообще** — ни одного изменения в
  `src/features/form-renderer/`, `pages/**/legacy-*`. Обе ветки switch'а
  `document-entry-page.tsx` работают как раньше.
- Бэкенд `talgat/SCRUM-268` на dev-стенде недоступен → проверка: unit-тесты чистых
  функций + `npm run build`; E2E-приёмка (§4 исходной спеки) — отдельно, когда бэк доедет.

## Архитектура

### 1. Общий хук опций (§3.1) + рефактор reference-field (§3.3)

**Новое:** `src/features/sdui/lib/hooks/use-reference-options.ts`

```ts
useReferenceOptions(
  fetcher: (search?: string) => Promise<SelectOption[]>,
  resetKey: string,
): {
  options: SelectOption[]
  loading: boolean
  load: (search?: string) => Promise<void>
  loadDebounced: (search: string) => void
  resetOptions: () => void
}
```

Внутрь переезжает inline-логика из reference-field: seq-гвард от гонок ответов,
debounce (300мс) с cleanup на unmount, сброс `options → []` по смене `resetKey`
(`useEffect([resetKey])`), тихое глотание ошибок fetch (UI показывает пустой список).
`resetOptions` — явная инвалидация кэша (нужна для сброса после выбора значения).

**`api/reference-options.ts`** остаётся источником fetch-функций (`fetchReferenceOptions`
уже соответствует `fetchEntryOptions` из спеки: `content ?? items`, маппинг в
`SelectOption`). Добавляются конверсии `referenceValueToOption` / `optionToReferenceValue`
(переезд из компонента).

**`reference-field-node.tsx`** — рефактор: удаляются `useState options/loading`,
`requestSeqRef`, `debounceRef`, локальные useEffect'ы; остаётся двухветочный fetcher
(`optionsSource` → его url/params; иначе legacy `/api/${domainPath}/${targetTypeCode}/entries`)
+ `useReferenceOptions`. Фикс кэша опций после выбора (`setOptions([])` в `applySelected`)
переезжает в вызов `resetOptions()`. Аффордансы/visible/actions-гейты не меняются.
`resetKey = optionsSource?.params ? JSON.stringify(params) : JSON.stringify(filter ?? null)`.

**Гейт рефактора:** существующий `reference-field-node.test.tsx` (сброс кэша → повторный
fetch на onOpen) проходит без изменения теста.

### 2. OBJECT_FIELD (§3.2)

**Новое:** `src/features/sdui/ui/nodes/composite/object-field-logic.ts` — чистые функции:

```ts
interface ObjectValue {
  id: number
  presentation: string
  type?: string          // DomainKind — только ПИШЕМ при эмите
  targetTypeCode: string // единственное поле, читаемое из входящего значения
}
interface AllowedType {
  position: number
  domainKind: string
  targetTypeCode: string
  presentation: string
  optionsSource?: { url: string; params?: Record<string, string> }
}

sortAllowedTypes(types)                                   // по position
resolveSelectedTypeCode(allowedTypes, value, userTypeCode)
// приоритет: value.targetTypeCode (если ∈ allowedTypes) → userTypeCode → allowedTypes[0]
findAllowedType(allowedTypes, typeCode)
buildObjectValue(member, option)  // { id, presentation, type: member.domainKind, targetTypeCode: member.targetTypeCode }
```

⚠️ Развилка имён (§2.5 контракта): DomainKind во входящем значении лежит под `domain`,
в исходящем — под `type`. Виджет НЕ читает ни то, ни другое из входящего значения —
только `targetTypeCode` (он один round-trip'ится и различает same-domain члены,
P0-урок «Владелец: Организации|Контрагенты — оба DICTIONARY»).

**Замена заглушки** `object-field-node.tsx`: горизонтальный ряд — MUI `Select` члена
(minWidth 160, label i18n `objectField.type`, пункты из `allowedTypes`, value =
`targetTypeCode`) + `AutocompleteInput` значения через `useReferenceOptions`
(fetcher по `optionsSource` выбранного члена; `resetKey = JSON.stringify(optionsSource ?? null)`;
`key={selectedTypeCode}` — перемонтаж пикера при смене члена). Проводка полей —
через `useFieldNode` (как у остальных полей: label/required/readonly/enabled/visible/error,
setValue, гейт EVENT `change` по `action{trigger:'change',actionId:'fieldEvent'}`).

Поведение:
- единственный локальный стейт — `userTypeCode` (ручной выбор члена при пустом значении);
  тип из значения всегда приоритетнее (селектор производный);
- смена члена: `setUserTypeCode(next)`; если значение было — `emitChange(null)`
  (семантика 1С: смена члена всегда чистит значение, без кэша «по типу»);
- выбор записи: `emitChange(buildObjectValue(member, option))`;
- `selectedOption` в пикере показывается только если `value?.targetTypeCode ===`
  выбранный член (иначе null);
- edge-cases: `allowedTypes` пуст → null; `visible === false` → null;
  `readonly`/`enabled=false` → селектор и пикер disabled/readOnly;
  член без `optionsSource` (примитив/ENUMS) → disabled `TextField` c placeholder
  i18n `objectField.unsupportedMember`, не падаем.

### 3. dialog-host — живое дерево (§3.4)

`src/features/sdui/ui/dialog-host.tsx`. Инфраструктура готова (`tree`-стейт,
`setRoot`, `applyTreePatches` в контексте панельной сессии) — баг в одном месте:
`DialogHost` рендерит `<NodeRenderer node={panel.node} />` (статичный снимок),
патчи мутируют `tree`, но рендер их не видит.

Фикс: `PanelFormProvider` сам рендерит `<NodeRenderer node={tree} />` из живого
стейта; внешний children-снимок в `DialogHost.map` убирается. Панели без сессии
(`panel.session == null`) — прежний путь (статики достаточно, патчей нет).
Заголовок панели — `panel.node.props?.title` только если `typeof === 'string'`.

### 4. download — attachment vs inline (§3.5)

`src/features/sdui/lib/effect-handler.ts`, case `'download'`:

- **Новое:** чистая функция `parseContentDispositionFilename(header): string` в
  `src/features/sdui/lib/parse-content-disposition.ts`: сперва RFC 5987
  `filename*=UTF-8''<pct-encoded>` (decodeURIComponent, обрезка кавычек; на кривой
  encoding — фолбэк на plain), затем `filename="…"`, иначе `''`.
- Ветвление по заголовку ответа `content-disposition` (axios-ответ `getFileBlob`
  отдаёт headers):
  - `/attachment/i` → `<a download={filename || 'download'} href={objectUrl}>` +
    `click()` + `remove()` — сохранение на диск;
  - иначе → `window.open(objectUrl, '_blank')` — inline-превью (как сейчас);
  - в обоих случаях `setTimeout(() => URL.revokeObjectURL(objectUrl), 60_000)`.
- Тост ошибки — через i18n (`sdui.downloadError`, ru «Не удалось сформировать файл»)
  вместо хардкода — закрывает LOW-находку ревизии по `effect-handler.ts:50`.

### 5. Роуты /documents/* (§3.6)

**`App.tsx`:** два новых роута:

```
/documents/:typeCode      → <DocumentRedirect mode="list" />
/documents/:typeCode/new  → <DocumentRedirect mode="new" />
```

**Новый слайс** `src/pages/documents/document-redirect/`:

- `lib/resolve-document-page-code.ts` — чистые:
  `moduleContainsType(items, typeCode)` (обход колонки → секции →
  `element.code === typeCode`) и
  `resolveDocumentPageCode(moduleCodes, itemsByModuleCode, typeCode)`
  (перебор в порядке сайдбара, первый модуль с типом, иначе `undefined`);
- `lib/use-resolve-document-page-code.ts` — хук:
  `useQuery(['settings','modules'])` (список кодов) + `useQueries` по
  `['settings','modules', code]` c `getModule(code)` — **те же queryKey, что у
  `useModule`** (кэш общий, открытые разделы не перезапрашиваются), `staleTime` 5 мин;
  `isResolving`, `pageCode`;
- `ui/document-redirect.tsx`: резолвится → `<Navigate replace>` на
  `/modules/${pageCode}/document/${typeCode}` (+ `/new` + `location.search` при
  `mode='new'` — `?copyFrom=…` сохраняется целиком); резолвится → `<PageSkeleton/>`;
  не найден → `console.warn` + `<Navigate to="/" replace />`.

Никакого хардкода тип→раздел.

## Отклонения от канона (ревизия 2026-07-02)

Все три навязаны бэкенд-контрактом SCRUM-268, зафиксированы в ревизии (раздел 9) как
временные мосты с условиями снятия:

1. **Роуты `/documents/*` + фронтовый резолв `pageCode`** — против «одного catch-all
   роута» и «маршрут целиком с бэка». Бэк не знает `pageCode` (shell-концепт фронта,
   сам помечен на перестройку). Снимается вместе с легаси-роутами при server-driven shell.
2. **Legacy-ветка `DOMAIN_PATH_MAP`** (поле без `optionsSource`) — против находки 2.1.
   SCRUM-268 §2.7 явно сохраняет legacy-путь. Не ухудшаем (код существующий, переезжает
   в fetcher). Снимается, когда бэк шлёт `optionsSource` всегда.
3. **`visible ?? true`, `enabled ?? true` у OBJECT_FIELD** — против решения №2
   (запрещающие дефолты). SCRUM-268 §2.6 фиксирует разрешающие; так работают все
   существующие поля. Снимается согласованной волной фронт+бэк.
