## Контекст

Документ `RegistratsiyaZayavleniyPoVychetamIPN`: две редактируемые ТЧ рядом (HSTACK) —
master `VychetyIPN` слева, detail `GrafikVycheta` справа. Связь: `masterTable="VychetyIPN"`,
`masterKey="VychetIPN"`, `detailKey="VychetIPN"`. Обе рендерятся `ComplexEditableTable`
(COLUMN_GROUP `orientation="VERTICAL"`).

Источник требований: спека бэкендера `frontend-spec-ipn-complex-tables-bugs.md`
(вложение SCRUM-282, копия в `specs-local/scrum-282-svyazannye-tablicy/`). Все 5 багов

table-EVENT, `showRowNumbers=true` на обеих TABLE, `masterTable` равен binding master-таблицы.

Все изменения — зона SDUI (`src/features/sdui/`), легаси не затрагивается.

## #4 — Связь таблиц не фильтрует (критично)

**Первопричина (b): нереактивное чтение выбора.**
`complex-editable-table.tsx:85-90` читает `getValue(masterTable + '.__selectedRowId')` и
`getValue(masterTable)` — разовый снимок zustand без подписки; при клике по master-строке
detail не ре-рендерится.

**Фикс:** читать оба значения через реактивный `useBindingValue`
(`sdui-session-context.tsx:64`); добавить выбранный id и строки master в deps `useMemo`
для `visibleRows`.

**Первопричина (c): ключ связи не проставляется в новую detail-строку.**
`buildEmptyRow` (`use-table-sync.ts:39`) ставит только дефолты по dataType; скрытая колонка
`VychetIPN` остаётся пустой — строка не матчится фильтром и не уходит с ключом на save.

**Фикс:** `sync.addRow(columns, presetValues?)` — опциональный второй параметр с
предустановленными значениями, мёржится поверх пустой строки. Detail-таблица передаёт
`{ [detailKey]: <значение masterKey выбранной master-строки> }`.

**Кнопка «Добавить» в detail:** блокируется, пока в master не выбрана строка
(решение согласовано; поведение как в 1С). `TableToolbar` получает признак недоступности Add.

**Не меняем:** когда в master ничего не выбрано, detail показывает все строки
(текущее поведение, спека это багом не помечала).

## #5 — После правок ТЧ save/post молча мертвы (критично)

`dispatch.ts:63` ждёт `flushAllPendingTableCommits()` до отправки COMMAND; `flushPending`
(`use-table-sync.ts`) резолвится только по приходу нового canon. Если canon не придёт —
save заблокирован навсегда, без ошибки.

**Фикс (guard):** в `flushAllPendingTableCommits` (`pending-table-commits.ts`) каждый flush
оборачивается в `Promise.race` с таймаутом ~5 сек. По таймауту flush считается завершённым,
save продолжается. Одно центральное изменение покрывает все таблицы. Ветка reject
(ошибка сети → тост `sdui.tableFlushFailed`) сохраняется — reject происходит раньше таймаута.
Бэк-половина (canon эхоится всегда) уже закрыта; guard — защита от будущих регрессий.

## #2 — Нет колонки «№»

`ComplexEditableTable` не рендерит колонку «№» (в отличие от `EditableTable` и `ReadOnlyTable`).
Бэк шлёт `showRowNumbers=true`.

**Фикс:** по флагу `showRowNumbers`: ячейка «№» в первом ряду шапки с `rowSpan` на все ряды
header groups (группированная шапка COLUMN_GROUP не разъезжается), `index + 1` в строках,
`+1` к `colSpan` пустого состояния. Образец — `editable-table.tsx:134`.

## #1 — Таблицы сжаты (HSTACK не управляет шириной)

`hstack-node.tsx` раскладывает детей `flex-row` без `flex`/`width` → TABLE ужимается до контента.

**Фикс:** каждый ребёнок HSTACK оборачивается в контейнер `flex: 1 1 0; min-width: 0` —
равное деление ширины. `min-width: 0` обязателен, иначе горизонтальный скролл всей формы.

**Риск:** HSTACK — общий layout, равное растягивание может задеть другие экраны (ряды кнопок,
`justify`). При реализации проверить использование; при конфликте — учитывать `props.flex`
ребёнка (дефолт 1). Позже ширины уедут в отдельный конфиг-сервис стилей (вне скоупа).

## #3 — Разная высота строк левой и правой таблиц

Две независимые MUI `<Table>`; VERTICAL-группа рендерит в ячейке стопку редакторов разной
интринсивной высоты (слева checkbox+text, справа date+date).

**Фикс:** единая константа `min-height` строки в `ComplexEditableTable` (обе таблицы пары —
этот компонент), достаточная для двухстрочной VERTICAL-ячейки. Точное значение подбирается
визуально. Позже уедет в конфиг-сервис стилей (вне скоупа).
