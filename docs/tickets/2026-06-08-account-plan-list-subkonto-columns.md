# План счетов: отдавать виды субконто (Субконто1/2/3) в ответе списка `/entries`

- **Тип:** доработка backend
- **Приоритет:** medium
- **Дата:** 2026-06-08
- **Экран:** «План счетов», `/modules/.../accountplan/{typeCode}` (домен `ACCOUNT_PLAN`),
  напр. `EdiniyPlanSchetovGosUchrezhdeniya`

## Цель

Привести колонки веб-«Плана счетов» к виду 1С. В 1С список счетов показывает,
помимо прочего, **Субконто1, Субконто2, Субконто3** (виды субконто счёта).
Сейчас в вебе этих колонок нет, и фронт не может их показать — данных нет в
ответе списка.

## Проблема (данные)

Эндпоинт списка:

```
GET /api/account-plan/{typeCode}/entries
```

Возвращает `AccountPlanEntryDto` БЕЗ информации о субконто. Виды субконто
доступны только отдельной ручкой по одному счёту:

```
GET /api/account-plan/entries/{id}/subkonto-kinds
```

Тянуть её на каждый счёт из грида (дерево в сотни счетов) — это N+1
запросов, неприемлемо. Поэтому показать Субконто1/2/3 в списке без доработки
backend нельзя.

## Что сделать

Добавить в каждый элемент ответа `/entries` краткую информацию о видах
субконто счёта — по позициям 1..3. Достаточно имён (то, что рисуется в
ячейке), без полной структуры `subkonto-kinds`.

Предлагаемая форма (минимально достаточная для грида):

```json
{
  "id": 104,
  "code": "1330",
  "nameRu": "Готовая продукция",
  "...": "...",
  "subkontoKinds": [
    { "position": 1, "kindNameRu": "Номенклатура",     "kindNameKz": null },
    { "position": 2, "kindNameRu": "Физические лица",   "kindNameKz": null }
  ]
}
```

Требования:
- массив отсортирован/читается по `position` (1..3); отсутствующие позиции
  просто не приходят (как у счёта с 2 субконто);
- `kindNameRu` обязателен, `kindNameKz` — если есть (иначе фронт фолбэчит на
  `kindNameRu`);
- у групп/счетов без субконто — пустой массив или отсутствие поля.

Это тот же набор, что уже отдаёт `/subkonto-kinds`
(`position`, `kindNameRu`, `kindNameKz`), просто инлайном в списке.

## Приёмка

- `/api/account-plan/{typeCode}/entries` для каждого счёта содержит виды
  субконто по позициям 1..3 (имена);
- на экране «План счетов» появляются колонки Субконто1 / Субконто2 /
  Субконто3, заполненные именами видов субконто, как в 1С — **без N+1
  запросов с фронта** (фронт берёт данные прямо из ответа списка).

## Контекст по остальным колонкам 1С (для справки, backend не требуется)

Эти 1С-колонки фронт может добавить сам — данные уже в `/entries`:

| 1С-колонка | Поле в `AccountPlanEntryDto` |
|---|---|
| Полное наименование (каз.) | `nameKz` (часто `null` → фолбэк `nameRu`) |
| Номер МО | `attributes['NomerMemorialnogoOrdera']` |
| Сс | `code1C` |

Бэкенд-доработка нужна **только** по субконто (этот тикет).
