# Журнал проводок госучреждения: кредитная аналитика показывала дебетовое значение

> Трек: движения документа / регистр бухгалтерии. **Фикс фронтовый. Бэк без изменений.**

## Финальное доказательство (стенд, перепроведённый документ)

Перемещение ТМЗ AAY00-00002 (внутреннее перемещение), форма документа:
- «Подразделение организации» (отправитель) = **Лаура**;
- «Подразделение получатель» = **Даурен**.

Правильная проводка Дт1315/Кт1315: Дт «Подразделение» = Даурен (приход/получатель),
**Кт «Подразделение» = Лаура** (расход/отправитель).

На гриде движений: Дт=Даурен (верно), **Кт=Даурен (неверно, должно быть Лаура)**.
Кредитный биндинг `podrazdelenieKt ?? Podrazdelenie` показал Даурен ⟹ поля
`podrazdelenieKt` в ответе accounting-группы REST `/id/movements` **НЕТ**
(иначе была бы Лаура; сработал фолбэк на одиночное `Podrazdelenie`=Даурен).

**Где Лаура ЕСТЬ:** в форме документа; в накопительном регистре `DvizheniyaTMZ`
(entry «Расход» → `PodrazdelenieOrganizatsii`=Лаура, id 262424); в SDUI-эндпоинте
`GET /api/view/movements/{id}` (childState `_podrazdelenieKt`=Лаура). **Где НЕТ:**
в accounting-группе REST `GET /api/document-entries/id/{id}/movements` — только
одиночное `Podrazdelenie`=Даурен.

⟹ Расхождение с разведкой снято: парные Дт/Кт-поля есть в **SDUI**-пайплайне
(childState) и не проброшены в **REST accounting-группу**, которую рендерит
полностраничный вид. Бэк УЖЕ считает обе стороны (они в SDUI childState) —
нужно лишь положить их в DTO accounting-группы REST.

## Первопричина (уточнено по факту)

Парные `*Dt`/`*Kt`-измерения бэк считает и отдаёт **в SDUI-пайплайне**
(`GET /api/view/movements/{id}` → childState `_podrazdelenieDt`=Даурен,
`_podrazdelenieKt`=Лаура; та же механика, что у `subkonto*Dt/Kt`). Но
полностраничный вид «Движения документа» рендерит **REST accounting-группу**
`GET /api/document-entries/id/{id}/movements`, а в ней измерения — **одиночным
коллапс-полем** `Podrazdelenie` (намеренно = Дт). Парных `podrazdelenieKt`/… там
НЕТ (доказано гридом: кредит показал Даурен через фолбэк, а не Лауру).

⟹ **Остаток — бэк, но узкий:** пробросить уже посчитанные Дт/Кт-стороны
измерений в DTO accounting-группы REST. Фронтовая часть (раздельные биндинги)
готова и включится сразу, как поля появятся.

## Фикс фронта (применён, готов)

`src/pages/documents/document-movements/ui/accounting-postings-table.tsx`:
- `ROW_FIELDS`: заголовок колонки (`*Label`) отделён от значений сторон
  (`*Dt`/`*Kt`); значения бэка — `podrazdelenieDt`/`podrazdelenieKt`,
  `fkrDt`/`fkrKt`, `spetsifikaDt`/`spetsifikaKt`,
  `istochnikFinansirovaniyaDt`/`istochnikFinansirovaniyaKt`,
  `kodPlatnykhUslugDt`/`kodPlatnykhUslugKt`; `kolichestvo` — общее на проводку;
- дебет: `entry[*Dt] ?? entry[*Label]`, кредит: `entry[*Kt] ?? entry[*Label]` —
  раздельное поле стороны, а если его нет (не перепроведено/одиночный контракт) —
  фолбэк на общий код, чтобы ячейки не пустовали. При наличии `*Kt` кредит
  показывает своё значение (подразделение Кт = Лаура), а не дебетовое.

Заголовок остаётся единым над Дт и Кт (как в 1С), значения — строго по сторонам.

**Фронтовый обходной путь для «Подразделение» (без бэка/SDUI):**
`deriveCreditDimensions` (`lib/derive-credit-dimensions.ts`) достаёт кредитное
подразделение из НАКОПИТЕЛЬНОГО регистра того же ответа `/movements` (entry с
`_movementKind === 'Расход'` → `PodrazdelenieOrganizatsii` = Лаура) и передаёт
в таблицу как `creditOverrides`. Приоритет: прямое `podrazdelenieKt` →
выведенный кредит → одиночное `Podrazdelenie`. Значение берётся только когда
однозначно (один расход), иначе — штатный фолбэк. Так Кт «Подразделение» = Лаура
уже сейчас, на существующих данных. Когда бэк добавит `podrazdelenieKt` —
приоритет сам переключится на него, обходной путь станет неактивным.

## Бэк (остаток — узкий)

REST accounting-группа `GET /api/document-entries/id/{id}/movements` в
`entries[]`/`columns[]` должна отдавать измерения **раздельно по сторонам**
(`fkrDt/fkrKt`, `podrazdelenieDt/podrazdelenieKt`, `spetsifikaDt/spetsifikaKt`,
`istochnikFinansirovaniyaDt/istochnikFinansirovaniyaKt`,
`kodPlatnykhUslugDt/kodPlatnykhUslugKt`) — так же, как уже отдаёт
`subkonto*Dt/Kt` и `account*`. Значения обеих сторон **уже посчитаны** и лежат в
SDUI childState (`_podrazdelenie*`), т.е. это проброс существующих данных в DTO
accounting-группы, не новый расчёт. Одиночное коллапс-`Podrazdelenie` (=Дт)
можно оставить для совместимости — фронт возьмёт `*Kt`, когда появится.
`kolichestvo`/`summa`/`soderzhanie`/`Organizatsiya` — общие, по разу.

## Приёмка

Перемещение ТМЗ AAY00-00002: Дт «Подразделение»=Даурен, **Кт «Подразделение»=Лаура**;
прочие парные измерения — свои значения по сторонам; количество/сумма/содержание
— по разу на проводку.
