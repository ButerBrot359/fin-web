# Журнал проводок госучреждения: кредитная аналитика показывала дебетовое значение

> Трек: движения документа / регистр бухгалтерии. **Фикс фронтовый. Бэк без изменений.**

## Итоговая первопричина (проверено на задеплоенном API)

Бэк **уже** отдаёт раздельные `*Dt`/`*Kt`-измерения проводки — той же
механикой, что у `subkonto*Dt/Kt`. Подтверждено на активной проводке
(Перемещение ТМЗ AAY00-00002, accounting-entry `_id 4438`):

- REST `GET /api/document-entries/id/{id}/movements` → `entries[]` содержит
  `podrazdelenieDt`=Даурен, `podrazdelenieKt`=Лаура, а также
  `fkrDt/fkrKt`, `spetsifikaDt/spetsifikaKt`,
  `istochnikFinansirovaniyaDt/istochnikFinansirovaniyaKt`,
  `kodPlatnykhUslugDt/kodPlatnykhUslugKt`;
- SDUI `GET /api/view/movements/{id}` → `columns[]` содержит парные колонки
  `_podrazdelenieDt`/`_podrazdelenieKt` (обе с title «Подразделение»),
  childState: `_podrazdelenieDt`=Даурен, `_podrazdelenieKt`=Лаура.

⚠️ Одиночное поле `AccountingRegisterEntryDto.podrazdelenie` (и одноимённые
`FKR`/`Spetsifika`/… без суффикса) — **коллапс-поле, намеренно = Дт**. Для
рендера сторон его использовать нельзя — это и сбило первичную диагностику
(разбирался стейл/коллапс-ответ, где парных полей не было видно).

**Реальный обрыв — на фронте:** обе колонки называются одинаково
«Подразделение» (как в 1С). Кастомный грид проводок читал ОДИН биндинг
(`a1`/`a2`) для обоих блоков → кредитный блок показывал дебетовое значение. Те
же грабли у пар ФКР/Специфика/Источник/КодПУ (не видны, когда стороны совпадают).

## Фикс (фронт, применён)

**Эндпоинт — без изменений:** `GET /api/document-entries/id/{id}/movements`
(вариант без `id/` groups-структуру не отдаёт → «Нет движений», откатан).
Раздельные `*Dt`/`*Kt` появляются в ответе этого же эндпоинта **после
перепроведения** документа (до перепроведения был одиночный контракт — отсюда
ранняя путаница).

`src/pages/documents/document-movements/ui/accounting-postings-table.tsx`:
- `ROW_FIELDS`: заголовок колонки (`*Label`) отделён от значений сторон
  (`*Dt`/`*Kt`); значения бэка — `podrazdelenieDt`/`podrazdelenieKt`,
  `fkrDt`/`fkrKt`, `spetsifikaDt`/`spetsifikaKt`,
  `istochnikFinansirovaniyaDt`/`istochnikFinansirovaniyaKt`,
  `kodPlatnykhUslugDt`/`kodPlatnykhUslugKt`; `kolichestvo` — общее на проводку;
- дебет: `entry[*Dt] ?? entry[*Label]`, кредит: `entry[*Kt] ?? entry[*Label]` —
  раздельное поле стороны, а если его нет (не перепроведено/одиночный контракт) —
  фолбэк на общий код, чтобы ячейки не пустовали. При наличии `*Kt` кредит
  показывает своё значение (подразделение Кт = Лаура), а не дебетовое.

Заголовок остаётся единым над Дт и Кт (как в 1С), значения — строго по сторонам.

## Бэк

Изменений не требуется — парные поля уже в контракте и задеплоены. Задачу
«добавить парные измерения на бэке» НЕ брать (уже сделано; повторная реализация
рискует сломать корректную выдачу).

## Приёмка

Перемещение ТМЗ AAY00-00002: Дт «Подразделение»=Даурен, **Кт «Подразделение»=Лаура**;
прочие парные измерения — свои значения по сторонам; количество/сумма/содержание
— по разу на проводку.
